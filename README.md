## 🤝 Contributing to Documentation

We welcome contributions to improve our documentation! Please see our [Contributing Guide](CONTRIBUTING.md) for:

- Documentation standards and style guide
- How to propose changes
- Review process for documentation updates
- Local development setup for docs

### Quick Contribution Guidelines

1. **Clarity First**: Write for your audience - developers, business users, or partners
2. **Keep It Current**: Update docs when features change
3. **Examples Matter**: Include code samples and real-world scenarios
4. **Link Liberally**: Cross-reference related documentation
5. **Test Your Changes**: Verify links and code examples work

## 📋 Documentation Versioning

- **Latest**: Always reflects the current main branch
- **Stable Releases**: Tagged versions available in releases
- **API Versions**: Documented in OpenAPI spec with version headers
